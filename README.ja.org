=ox-hub= は、1つの Org-mode 原稿から Zenn と Qiita の両方に向けた
Markdown を生成する Emacs package です。

MVP では =org/<slug>.org= を正本として扱い、次のファイルを生成します。

- =dist/zenn/articles/<slug>.md=
- =dist/qiita/public/<slug>.md=

公開、Git push、Zenn CLI、Qiita CLI、リモート投稿状態との同期は v1 の範囲外です。

* Installation

この repository を clone し、=load-path= に追加します。

#+begin_src emacs-lisp
(add-to-list 'load-path "/path/to/ox-hub")
(require 'ox-hub)
#+end_src

現時点では source checkout での利用を想定しています。package archive からの
installation はまだ提供していません。

* Commands

- =ox-hub-new-article=
  - Git root 配下に =org/<slug>.org= を作成します。
  - slug は =a-z0-9_-= の 12 文字以上 50 文字以下です。
  - 既存ファイルがある場合は拒否します。
  - 作成後、新しい記事ファイルを開きます。

- =ox-hub-export-current-buffer=
  - 現在の Org buffer を Zenn と Qiita の Markdown に export します。
  - 出力ファイルは上書きします。

- =ox-hub-export-current-buffer-to-zenn=
  - 現在の Org buffer を Zenn Markdown のみに export します。

- =ox-hub-export-current-buffer-to-qiita=
  - 現在の Org buffer を Qiita Markdown のみに export します。

* Quick Start

対象 Git repository 内の file visiting buffer から新しい記事を作成します。

#+begin_src text
M-x ox-hub-new-article
#+end_src

=my-first-note= のような slug を入力すると、次のファイルを作成して開きます。

#+begin_src text
org/my-first-note.org
#+end_src

=OXHUB_TITLE= と =OXHUB_TAGS= を埋め、本文を書いてから export します。

#+begin_src text
M-x ox-hub-export-current-buffer
#+end_src

この command は次のファイルを書き出します。

#+begin_src text
dist/zenn/articles/my-first-note.md
dist/qiita/public/my-first-note.md
#+end_src

* End-to-end Example

Org 原稿:

#+begin_src org
#+OXHUB_TITLE: Example Title
#+OXHUB_TAGS: emacs, org-mode
#+OXHUB_STATUS: draft
#+OXHUB_ZENN_EMOJI: 📝
#+OXHUB_ZENN_TYPE: tech
#+OXHUB_QIITA_PRIVATE: false
#+OXHUB_QIITA_SLIDE: false

,* Hello

This is *bold* and =code=.

#+begin_oxhub message :type alert
Be careful.
#+end_oxhub
#+end_src

Zenn output:

#+begin_src markdown
---
title: "Example Title"
emoji: "📝"
type: "tech"
topics: ["emacs", "org-mode"]
published: false
---

# Hello

This is **bold** and `code`.

:::message alert
Be careful.
:::
#+end_src

Qiita output:

#+begin_src markdown
---
title: "Example Title"
tags:
  - "emacs"
  - "org-mode"
private: false
slide: false
ignorePublish: true
---

# Hello

This is **bold** and `code`.

> **Warning:**
>
> Be careful.
#+end_src

* Metadata

記事 metadata は =#+OXHUB_*:= keyword で書きます。

必須項目:

#+begin_src org
#+OXHUB_TITLE:
#+OXHUB_TAGS:
#+OXHUB_STATUS:
#+OXHUB_ZENN_EMOJI:
#+OXHUB_ZENN_TYPE:
#+OXHUB_QIITA_PRIVATE:
#+OXHUB_QIITA_SLIDE:
#+end_src

=OXHUB_STATUS= は =draft= または =published= です。

status は次のように出力されます。

| OXHUB_STATUS | Zenn =published= | Qiita =ignorePublish= |
|--------------+------------------+-----------------------|
| =draft=      | =false=          | =true=                |
| =published=  | =true=           | =false=               |

=OXHUB_TAGS= は comma-separated list です。空 tag は無視され、export 時には
1つ以上の tag が必要です。

boolean metadata は =true=, =false=, =t=, =nil= を受け付けます。

新規記事は次の雛形で作成されます。

#+begin_src org
#+OXHUB_TITLE:
#+OXHUB_TAGS:
#+OXHUB_STATUS: draft
#+OXHUB_ZENN_EMOJI: 📝
#+OXHUB_ZENN_TYPE: tech
#+OXHUB_QIITA_PRIVATE: false
#+OXHUB_QIITA_SLIDE: false
#+end_src

* Supported Org Syntax

共通 renderer は次の Org 記法を扱います。

| Org syntax                  | Markdown output                         |
|-----------------------------+-----------------------------------------|
| =*=, =**= headings          | =#=, =##= headings                      |
| paragraphs                  | paragraphs                              |
| unordered / ordered lists   | Markdown lists, including nested lists  |
| =[[url][text]]=             | =[text](url)=                           |
| =[[file:image.png]]=        | =![](image.png)=                        |
| =[[file:notes.org][Notes]]= | =[Notes](notes.org)=                    |
| =#+begin_quote=             | blockquote                              |
| Org tables                  | Markdown tables                         |
| =#+begin_src lang=          | fenced code block                       |
| =#+begin_example=           | fenced code block without language      |
| ==code== / =~code~=         | inline code                             |
| =*bold*=                    | bold                                    |
| =/italic/=                  | italic                                  |
| =+strike+=                  | strikethrough                           |
| =-----=                     | horizontal rule                         |
| =[fn:label]=                | footnote reference                      |
| =[fn:label] body=           | footnote definition                     |
| =#+begin_src mermaid=       | fenced =mermaid= block                  |

未対応の Org 要素に遭遇した場合、黙って落とさず export を失敗させます。

* oxhub Directives

Zenn/Qiita 固有の本文記法は Org special block で表現します。

#+begin_src org
#+begin_oxhub message :type info
Informational message.
#+end_oxhub
#+end_src

=:type= は =info= または =alert= です。省略時は =info= です。

Zenn output:

#+begin_src markdown
:::message
Informational message.
:::
#+end_src

Qiita output:

#+begin_src markdown
> Informational message.
#+end_src

alert message は Zenn では =:::message alert=、Qiita では =**Warning:**= 付きの
blockquote になります。

details block には =:summary= が必要です。

#+begin_src org
#+begin_oxhub details :summary "More details"
Hidden body.
#+end_oxhub
#+end_src

Zenn output は =:::details= を使います。Qiita output は =<details>= HTML block
を使います。

ファイル名付き code block には =:lang= と =:filename= が必要です。

#+begin_src org
#+begin_oxhub codefile :lang emacs-lisp :filename init.el
(message "hello")
#+end_oxhub
#+end_src

どちらの target でも次のように出力します。

#+begin_src markdown
```emacs-lisp:init.el
(message "hello")
```
#+end_src

未対応の =oxhub= directive、不正な message type、必須 parameter の欠落は
export error になります。

* Limitations

v1 package は Zenn CLI、Qiita CLI、preview、publish、Git push、リモート投稿状態との
同期を実行しません。

asset copy は行いません。=[[file:...]]= link はそのまま Markdown link または
image reference に変換されます。

=message=, =details=, =codefile= 以外の platform-specific syntax はまだ未実装です。
例として、Zenn の画像幅/キャプション helper、Zenn embed、platform-specific diff
helper、PlantUML、Qiita color chip、Qiita HTML extension などがあります。
