=ox-hub= is an Emacs package for writing one Org-mode source article and
exporting Markdown for both Zenn and Qiita.

The MVP treats =org/<slug>.org= as the source of truth and generates:

- =articles/<slug>.md=
- =public/<slug>.md=

Publishing, Git push, Qiita CLI, and synchronization with remote posting state
are outside the v1 scope.  Zenn CLI can read the generated =articles/= and
=images/= directories from the repository root.

* Installation

Clone this repository and add it to =load-path=:

#+begin_src emacs-lisp
(add-to-list 'load-path "/path/to/ox-hub")
(require 'ox-hub)
#+end_src

This is a source checkout workflow. Package archive installation is not
provided yet.

* Commands

- =ox-hub-new-article=
  - Create =org/<slug>.org= under the Git root.
  - The Git root is detected from the current file, or from
    =default-directory= for buffers such as Dired.
  - The slug must match =a-z0-9_-= and be 12 to 50 characters long.
  - Existing files are rejected.
  - The new article is opened after creation.

- =ox-hub-export-current-buffer=
  - Export the current Org buffer to both Zenn and Qiita Markdown.
  - Outputs are overwritten.

- =ox-hub-export-current-buffer-to-zenn=
  - Export the current Org buffer to Zenn Markdown only.

- =ox-hub-export-current-buffer-to-qiita=
  - Export the current Org buffer to Qiita Markdown only.

* Quick Start

Create a new article from a buffer whose current file or =default-directory=
is inside the target Git repository:

#+begin_src text
M-x ox-hub-new-article
#+end_src

Then enter a slug such as =my-first-note=. The command creates and opens:

#+begin_src text
org/my-first-note.org
#+end_src

Fill in =OXHUB_TITLE= and =OXHUB_TAGS=, write the body, then export:

#+begin_src text
M-x ox-hub-export-current-buffer
#+end_src

The command writes:

#+begin_src text
articles/my-first-note.md
public/my-first-note.md
#+end_src

* End-to-end Example

Source Org article:

#+begin_src org
#+OXHUB_TITLE: Example Title
#+OXHUB_TAGS: emacs, org-mode
#+OXHUB_STATUS: draft
#+OXHUB_ZENN_EMOJI: 📝
#+OXHUB_ZENN_TYPE: tech
#+OXHUB_QIITA_PRIVATE: false
#+OXHUB_QIITA_SLIDE: false

,* Hello

This is *bold* and =code=.

#+begin_oxhub message :type alert
Be careful.
#+end_oxhub
#+end_src

Zenn output:

#+begin_src markdown
---
title: "Example Title"
emoji: "📝"
type: "tech"
topics: ["emacs", "org-mode"]
published: false
---

# Hello

This is **bold** and `code`.

:::message alert
Be careful.
:::
#+end_src

Qiita output:

#+begin_src markdown
---
title: "Example Title"
tags:
  - "emacs"
  - "org-mode"
private: false
slide: false
ignorePublish: true
---

# Hello

This is **bold** and `code`.

> **Warning:**
>
> Be careful.
#+end_src

* Metadata

Article metadata is written with =#+OXHUB_*:= keywords.

Required fields:

#+begin_src org
#+OXHUB_TITLE:
#+OXHUB_TAGS:
#+OXHUB_STATUS:
#+OXHUB_ZENN_EMOJI:
#+OXHUB_ZENN_TYPE:
#+OXHUB_QIITA_PRIVATE:
#+OXHUB_QIITA_SLIDE:
#+end_src

=OXHUB_STATUS= is either =draft= or =published=.

The status is rendered as:

| OXHUB_STATUS | Zenn =published= | Qiita =ignorePublish= |
|--------------+------------------+-----------------------|
| =draft=      | =false=          | =true=                |
| =published=  | =true=           | =false=               |

=OXHUB_TAGS= is a comma-separated list. Empty tags are ignored, and at least
one tag is required when exporting.

Boolean metadata accepts =true=, =false=, =t=, and =nil=.

New articles start with:

#+begin_src org
#+OXHUB_TITLE:
#+OXHUB_TAGS:
#+OXHUB_STATUS: draft
#+OXHUB_ZENN_EMOJI: 📝
#+OXHUB_ZENN_TYPE: tech
#+OXHUB_QIITA_PRIVATE: false
#+OXHUB_QIITA_SLIDE: false
#+end_src

* Supported Org Syntax

The common renderer supports:

| Org syntax                  | Markdown output                         |
|-----------------------------+-----------------------------------------|
| =*=, =**= headings          | =#=, =##= headings                      |
| paragraphs                  | paragraphs                              |
| unordered / ordered lists   | Markdown lists, including nested lists  |
| =[[url][text]]=             | =[text](url)=                           |
| =[[file:image.png]]=        | =![](image.png)=                        |
| =[[file:notes.org][Notes]]= | =[Notes](notes.org)=                    |
| =#+begin_quote=             | blockquote                              |
| Org tables                  | Markdown tables                         |
| =#+begin_src lang=          | fenced code block                       |
| =#+begin_example=           | fenced code block without language      |
| ==code== / =~code~=         | inline code                             |
| =*bold*=                    | bold                                    |
| =/italic/=                  | italic                                  |
| =+strike+=                  | strikethrough                           |
| =-----=                     | horizontal rule                         |
| =[fn:label]=                | footnote reference                      |
| =[fn:label] body=           | footnote definition                     |
| =#+begin_src mermaid=       | fenced =mermaid= block                  |

Unsupported Org elements fail the export instead of being silently dropped.

* oxhub Directives

Zenn/Qiita-specific body syntax is represented with Org special blocks:

#+begin_src org
#+begin_oxhub message :type info
Informational message.
#+end_oxhub
#+end_src

=:type= may be =info= or =alert=. =info= is the default.

Zenn output:

#+begin_src markdown
:::message
Informational message.
:::
#+end_src

Qiita output:

#+begin_src markdown
> Informational message.
#+end_src

Alert messages become =:::message alert= for Zenn and a blockquote with
=**Warning:**= for Qiita.

Details blocks require =:summary=:

#+begin_src org
#+begin_oxhub details :summary "More details"
Hidden body.
#+end_oxhub
#+end_src

Zenn output uses =:::details=. Qiita output uses a =<details>= HTML block.

File-name code blocks require =:lang= and =:filename=:

#+begin_src org
#+begin_oxhub codefile :lang emacs-lisp :filename init.el
(message "hello")
#+end_oxhub
#+end_src

Both targets output:

#+begin_src markdown
```emacs-lisp:init.el
(message "hello")
```
#+end_src

Unsupported =oxhub= directives, invalid message types, and missing required
parameters fail the export.

* Limitations

The v1 package intentionally does not run Zenn CLI, Qiita CLI, preview,
publish, Git push, or remote posting-state synchronization.

Assets are not copied. =[[file:...]]= links are converted to Markdown links or
image references as-is.

Platform-specific syntax beyond =message=, =details=, and =codefile= is not
implemented yet. Examples include Zenn image width/caption helpers, Zenn
embeds, platform-specific diff helpers, PlantUML, Qiita color chips, and Qiita
HTML extensions.
