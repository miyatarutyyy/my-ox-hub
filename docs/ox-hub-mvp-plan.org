#+TITLE: ox-hub MVP Implementation Plan
#+AUTHOR: ox-hub maintainers
#+LANGUAGE: ja

* Summary

=ox-hub.el= の MVP では、Org-mode 原稿を正本として扱い、Zenn と Qiita 向け Markdown を生成する。

変換は =ox-md= を使わず、Org AST を一度生成して自前 renderer で直接 Markdown 化する。本文 renderer は target を引数に取る単一のオーケストレータとし、通常の共通記法は共通 renderer で処理する。=ox-hub= 独自記法に遭遇した場合のみ、target に応じて Zenn/Qiita 用 renderer を呼び分ける。

v1 の担当範囲は「記事雛形の作成」と「Markdown 生成」までとする。Git push、GitHub Actions、Qiita CLI 実行、投稿状態同期は v1 の =ox-hub.el= では扱わない。

* Public Commands

v1 では以下の interactive command を提供する。

- =ox-hub-new-article=
  - Git root 配下の =org/<slug>.org= を作成する。
  - =<slug>= は =a-z0-9_-= の 12 文字以上 50 文字以下に制限する。
  - 既存ファイルがある場合は error にする。
  - 必須 =OXHUB_*= metadata の雛形を挿入する。

- =ox-hub-export-current-buffer=
  - 現在の Org buffer から Zenn/Qiita 両方の Markdown を生成する。
  - 出力先は =articles/<slug>.md= と =public/<slug>.md= とする。
  - =<slug>= は現在の Org ファイル basename から取得する。
  - 生成済み Markdown は常に上書きする。

- =ox-hub-export-current-buffer-to-zenn=
  - 現在の Org buffer から Zenn 用 Markdown だけを生成する。

- =ox-hub-export-current-buffer-to-qiita=
  - 現在の Org buffer から Qiita 用 Markdown だけを生成する。

以下の場合は変換を中断して error にする。

- 現在の buffer が file visiting buffer ではない。
- Git root が見つからない。
- Org ファイル名由来の slug が不正である。
- 必須 metadata が欠落している。
- metadata の値が不正である。
- 未対応 Org AST 要素に遭遇した。

* Repository Layout

記事リポジトリの root は、現在の Org ファイルから見て最寄りの =.git= が存在するディレクトリとする。

想定する構成:

#+begin_src text
.
|-- org/
|   `-- example-slug.org
|-- articles/
|   `-- example-slug.md
|-- images/
|-- public/
|   `-- example-slug.md
|-- src/
`-- .gitignore
#+end_src

=org/= はユーザーが編集する正本を置く。=articles/= と =public/= は生成物であり、手編集しない前提とする。=images/= は Zenn CLI が repository root から読む画像実体を置く。

* Metadata Specification

Org 側の front matter 入力は =#+OXHUB_*:= keyword に統一する。

v1 の必須項目:

#+begin_src org
#+OXHUB_TITLE:
#+OXHUB_TAGS:
#+OXHUB_ZENN_EMOJI:
#+OXHUB_ZENN_TYPE:
#+OXHUB_ZENN_PUBLISHED:
#+OXHUB_QIITA_PRIVATE:
#+OXHUB_QIITA_SLIDE:
#+OXHUB_QIITA_IGNORE_PUBLISH:
#+end_src

基本方針:

- =OXHUB_TITLE= は Zenn/Qiita の =title= に使う。
- =OXHUB_TAGS= は comma-separated で読み取り、Zenn の =topics= と Qiita の =tags= に変換する。
- =OXHUB_ZENN_EMOJI= は Zenn の =emoji= に使う。
- =OXHUB_ZENN_TYPE= は Zenn の =type= に使う。値は =tech= または =idea= とする。
- =OXHUB_ZENN_PUBLISHED= は Zenn の =published= に使う。
- =OXHUB_QIITA_PRIVATE= は Qiita の =private= に使う。
- =OXHUB_QIITA_SLIDE= は Qiita の =slide= に使う。
- =OXHUB_QIITA_IGNORE_PUBLISH= は Qiita の =ignorePublish= に使う。

boolean 値は =true=, =false=, =t=, =nil= を受け付け、内部では Lisp boolean に正規化する。不正値は error にする。

Qiita CLI が管理する =id=, =updated_at=, =organization_url_name= は v1 では Org metadata として扱わず、出力もしない。将来必要になった段階で、Org metadata として管理するか、既存 =dist/= から引き継ぐかを再設計する。

* Front Matter Rendering

Zenn 用 front matter は固定順で YAML を出力する。

#+begin_src yaml
---
title: "Example Title"
emoji: "📝"
type: "tech"
topics: ["emacs", "org-mode"]
published: false
---
#+end_src

Qiita 用 front matter も固定順で YAML を出力する。

#+begin_src yaml
---
title: "Example Title"
tags:
  - emacs
  - org-mode
private: false
slide: false
ignorePublish: false
---
#+end_src

YAML 文字列は renderer が生成し、metadata parser は plist などの構造化データだけを返す。

* Body Rendering

本文は =org-element-parse-buffer= で AST を取得し、自前 renderer で Markdown へ変換する。

v1 で対応する Org 要素:

- heading
- paragraph
- bold
- italic
- code
- src block / code block
- unordered list
- ordered list
- link
- quote
- table

未対応要素に遭遇した場合は warning ではなく error にする。

画像や添付ファイルはコピーしない。=[[file:...]]= は Markdown の参照リンクまたは画像リンクに変換するだけとする。画像アップロード、=dist/= への asset copy、投稿先別 asset 管理は v1 では扱わない。

* Structured Directives

Zenn/Qiita で差異のある本文記法は、Org の special block として =#+begin_oxhub ... #+end_oxhub= を使って表現する。

v1 で扱う directive:

- =message=
  - 補足・警告などのメッセージブロックを表す。
  - 例: =#+begin_oxhub message :type info=
  - =:type= は =info= または =alert= を想定する。

- =details=
  - 折りたたみブロックを表す。
  - 例: =#+begin_oxhub details :summary "詳細"=

- =codefile=
  - ファイル名付き code block を表す。
  - 例: =#+begin_oxhub codefile :lang emacs-lisp :filename init.el=

directive の内容も AST として処理し、AST 走査中にその場で target 別 renderer へ渡す。独自記法を Markdown 生成後の後処理で結合する設計にはしない。

* Error Policy

MVP では曖昧な変換を避けるため、次の状況を error とする。

- 必須 metadata の欠落。
- boolean や Zenn type の不正値。
- slug の不正値。
- 未対応 Org 要素。
- 未対応 =oxhub= directive。
- directive の必須引数欠落。

この方針により、生成された Markdown が投稿先で意図しない表示になる前に、Org 変換段階で失敗させる。

* Test Plan

テストは Emacs 標準の ERT で書く。

対象:

- =OXHUB_*= metadata の抽出。
- boolean 正規化。
- 必須項目 validation。
- slug validation。
- Zenn front matter の YAML 出力。
- Qiita front matter の YAML 出力。
- heading, paragraph, inline style, code block, list, link, quote, table の Markdown 出力。
- =message=, =details=, =codefile= directive の target 別出力。
- =[[file:...]]= が asset copy なしで Markdown link/image になること。
- Git root 不在、未保存 buffer、未対応 AST 要素、metadata 欠落が error になること。

想定 test command:

#+begin_src shell
emacs --batch -l ert -l ox-hub.el -l test/ox-hub-test.el -f ert-run-tests-batch-and-exit
#+end_src

将来的には =make test= で上記を実行できるようにする。

* Assumptions

- =dist/qiita/public<slug>.md= という表記は =dist/qiita/public/<slug>.md= の意図として扱う。
- =dist/= 配下の Markdown は再生成可能な成果物であり、手編集しない。
- GitHub Actions で Qiita CLI が front matter を更新しても、v1 ではその変更を repository へ自動で戻さない。
- Qiita の =id= や =updated_at= が必要になった場合は、v1 後に別途同期設計を追加する。
- Zenn/Qiita の CLI 実行、preview、publish 補助は今回の MVP には含めない。
