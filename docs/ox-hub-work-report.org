* Ticket 2: Metadata and Slug Validation

追加ファイル:

- =ox-hub.el=
- =test/ox-hub-test.el=

実装内容:

- =ox-hub--valid-slug-p=
  - =a-z0-9_-= のみ許可。
  - 12 文字以上 50 文字以下を valid とする。
  - Emacs の =case-fold-search= 影響を避けるため、slug validation 内では case-sensitive に固定。

- =ox-hub--git-root=
  - file visiting buffer または指定 file を起点に =.git= を探索。
  - 未保存 buffer または Git root 不在の場合は =user-error=。

- =ox-hub--parse-buffer=
  - =org-element-parse-buffer= により現在の Org buffer から AST を抽出。

- =ox-hub--extract-metadata=
  - Org AST の =keyword= 要素から =OXHUB_*= metadata を抽出。
  - keyword 名は大文字化して扱う。
  - 内部表現は plist。
  - duplicate metadata は現時点では後勝ち。

- =ox-hub--parse-boolean=
  - =true= / =t= を Lisp boolean の =t= に正規化。
  - =false= / =nil= を Lisp boolean の =nil= に正規化。
  - 大文字小文字は無視。
  - 不正値は error。

- =ox-hub--validate-metadata=
  - 必須 metadata の存在と空値を検証。
  - =OXHUB_STATUS= は =draft= / =published= のみ許可。
  - =OXHUB_ZENN_TYPE= は =tech= / =idea= のみ許可。
  - Qiita boolean fields を Lisp boolean に正規化。

* Current Design Notes

現在の設計認識:

- public command は最終的に正本 Org buffer に対する export entrypoint になる。
- 内部では =org-element-parse-buffer= で AST を抽出する。
- front matter は metadata 全体に対して 1 回 renderer を適用する。
- 本文は AST node ごとに dispatcher が処理を振り分ける。
- 通常 Org 要素は共通 Markdown converter に渡す。
- =oxhub= special block のみ target-specific converter に渡す。
